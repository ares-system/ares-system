# Product Requirements Document (PRD): ASST CLI Terminal Agent

## 1. Executive Summary
ASST is shifting from a web-based dashboard to a CLI-native "Agentic Tool" designed for high-performance Solana SecOps. This tool, modeled after "Claude Code," provides a persistent, stateful terminal interface for codebase analysis, real-time security scanning (L1-L6), and autonomous remediation.

## 2. Targeted Users
- **Solana Protocol Developers:** Seeking rapid security feedback during local development.
- **Security Auditors:** Requiring a unified interface for multi-lane repository analysis.
- **DevSecOps Engineers:** Automating security checks in local and CI environments.

## 3. Key Functional Requirements

### 3.1 CLI Core Infrastructure
- **Entry Point:** A global `asst` binary implemented via Commander.js.
- **TUI (Terminal User Interface):** Interactive prompts using `@clack/prompts` with an ASST-branded theme.
- **Persistence:** Local SQLite database per repository (`.asst/asst.db`) to store chat history and indexing metadata.

### 3.2 Agentic REPL (`asst chat`)
- **Persistent Memory:** The agent must remember previous turns within the same repository context.
- **Autonomous Tools:**
  - `read_file`: Deep analysis of Rust/TypeScript logic.
  - `write_file` (Safe Mode): Propose and apply code fixes with manual user confirmation.
  - `run_terminal_cmd`: Execute local tests (e.g., `anchor test`) or formatters.

### 3.3 Security Pipeline (`asst scan`)
- **6-Lane Assurance:** Porting existing logic to run sequentially in the terminal:
  - L1: Program Logic (Semgrep)
  - L2: Math/Precision
  - L3: Chain State (Account Analyzer)
  - L4: Access Control
  - L5: Asset Safety
  - L6: Supply Chain (pnpm audit)
- **Reporting:** Immediate findings output followed by an interactive remediation phase.

### 3.4 Semantic Search & Indexing
- **Indexing:** `asst index` command to crawl the repository and store content hashes.
- **Context Retrieval:** Agent can use `search_code` to find relevant files across large monorepos.

## 4. Technical Stack
- **Runtime:** Node.js (v20+), TypeScript.
- **AI Framework:** LangChain / LangGraph for stateful orchestration.
- **CLI Tools:** Commander, Clack, Chalk, Boxen, Ora.
- **Database:** SQLite3.
- **Indexing:** glob, crypto.

## 5. Security & Safety
- **Safe Mode:** Destructive tools (`write_file`, `run_terminal_cmd`) strictly require a TUI `confirm()` prompt.
- **Environment Isolation:** Local execution ensures no private code leaves the user's machine (except for LLM inference calls).
