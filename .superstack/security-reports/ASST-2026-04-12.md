## Findings (≥ 8/10 confidence)

### [HIGH] FINDING-001: Transitive vulnerable axios via @daytonaio/sdk

**Confidence:** 9/10  
**Phase:** Phase 3 — Dependency supply chain  
**Category:** OWASP A03 / A05 (depending on SDK usage)  
**Location:** `deepagentsjs/libs/providers/daytona/package.json` → `@daytonaio/sdk` → `axios` &lt; 1.15.0  

**Description:**  
`pnpm audit` reports two **critical** advisories on `axios` versions below 1.15.0, reachable through the Daytona provider dependency tree. Impact is typically SSRF and metadata-exfiltration class issues when axios is used with attacker-influenced URLs/headers in vulnerable patterns.

**Exploit scenario:**  
An application using the Daytona sandbox provider could remain exposed to axios CVE behavior until the dependency is updated or overridden, if the SDK passes user-influenced data into HTTP clients in affected code paths.

**Evidence:**  
`pnpm audit` output listing paths `libs__providers__daytona>@daytonaio/sdk>axios` and GHSA-3p68-rc4w-qgx5, GHSA-fvcv-3m26-pcqx.

**Remediation:**  
1. Prefer upgrading `@daytonaio/sdk` to a release that depends on `axios >= 1.15.0`.  
2. If upstream lags, add a **pnpm override** at monorepo root: `"axios": "^1.15.0"` (verify compatibility with SDK).  
3. Re-run `pnpm audit` and integration tests for Daytona.

**Status (follow-up):** Monorepo root [`deepagentsjs/package.json`](../../deepagentsjs/package.json) applies **`pnpm.overrides`** `"axios@<1.15.0": "1.15.0"`. As of this follow-up, **`pnpm audit`** reports **zero** vulnerabilities for `deepagentsjs/`. Upgrading `@daytonaio/sdk` beyond `^0.155.0` should be revisited when changelog/API compatibility is confirmed (newer SDK minors may change client surface).

**Priority:** P1 (fix this sprint)
